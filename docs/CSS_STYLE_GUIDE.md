# CSS 编码与注释规范

本文档旨在统一项目中的 CSS 代码风格，确保代码的可读性、可维护性和团队协作的一致性。所有 CSS 代码都应遵循以下规范。

---

## 1. 代码排版规范

所有 CSS 代码的格式化应严格遵循 Google CSS 编码规范，以保证代码的整洁与统一。

- **缩进 (Indentation):**
  - 每个缩进层级统一使用 **4 个空格**。

- **换行 (Line Breaks):**
  - 每个独立的 CSS 规则（选择器 + 声明块）之后必须保留 **一个空行**。
  - 每个属性声明（`property: value;`）都必须 **独占一行**。

- **空格 (Spacing):**
  - 在选择器与左大括号 `{` 之间保留 **一个空格**。
  - 属性名与冒号 `:` 之间 **无空格**。
  - 冒号 `:` 与属性值之间保留 **一个空格**。

- **示例:**
  '''css
  /* 正确的排版示例 */
  .selector-1,
  .selector-2 {
      padding: 1rem 1.5rem;
      font-size: 1rem;
      background-color: #007bff;
  }
  '''

---

## 2. 注释规范

清晰、统一的注释是代码可读性的关键。

- **注释语言:**
  - 所有注释都必须使用 **中文** 编写。

- **注释格式:**
  - 统一使用单行块注释 `/* 中文注释 */` 的格式。
  - 避免使用连续的多行注释；每个注释都应是独立的 `/* ... */` 单元。

- **注释位置:**
  - 注释必须 **独占一行**，并放置在所描述代码块的 **正上方**。
  - **严禁** 将注释写在代码行的末尾（行内注释）。

- **正确示例:**
  '''css
  /* 模态框头部的标题 */
  .modal-header h2 {
      margin: 0;
      font-size: 1.25rem;
  }
  '''

- **错误示例:**
  '''css
  /* 这是一个
     多行注释，不符合规范 */
  .modal-header h2 {
      margin: 0; /* 错误的行内注释 */
      font-size: 1.25rem;
  }
  '''
